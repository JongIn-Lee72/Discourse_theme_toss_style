# Toss Style Theme (토스 스타일 디스코스 테마)

**Toss Style Theme**는 Discourse 포럼의 UI/UX를 **토스 디자인 시스템(TDS)** 규격 및 **토스 테크 블로그(toss.tech)** 스타일로 전면 리디자인한 고품질 소셜 테마입니다. 기존의 복잡한 레이아웃과 선 위주의 그리드를 배제하고, 은은한 면 분할과 맑은 입체 그림자, 그리고 부드러운 인터랙션을 중심으로 고급스럽고 모던한 룩앤필을 제공합니다.

---

## :rocket: 핵심 디자인 및 레이아웃 개편 사양

### 1. TDS 디자인 토큰 및 믹스인 적용
- **표준 곡률 (Corner Radii)**: 임의의 모서리 둥글기 값을 배제하고, TDS 규격에 따라 8px(소형), 12px(중형 - 인풋, 버튼, 드롭다운), 18px(대형 - 피드 카드, 배너), 24px(초대형 - 모달 다이얼로그)로 일관성 있게 통일했습니다.
- **맑은 입체 그림자 (Shadows)**: 기존의 답답한 회색 테두리선(border)을 걷어내고, 토스 특유의 맑고 투명하며 깊이감 있는 은은한 드롭 섀도우(`box-shadow`) 스케일을 적용했습니다.
- **상태 레이어 피드백 (State Layers)**: 링크 및 카드 요소에 호버/액티브 시 부드럽게 반응하는 Toss 표준 감도 피드백을 적용했습니다.

### 2. 가로형 TDS 알약 칩 내비게이션 & 2열 와이드 그리드
- **2열 레이아웃 최적화**: 기존 3열 그리드에서 좌측 내비게이션 전용 컬럼을 제거하고, 피드 및 글 목록이 화면 좌측 끝단부터 시원하게 차오르는 와이드한 **2열 그리드(메인 피드 + 우측 프로필 패널)** 구조로 개편했습니다.
- **가로형 알약 칩(Pill Tabs) 탭**: 세로로 길게 배치되었던 필터 내비게이션 바를 환영 배너 아래의 가로형 알약형 칩 형태로 정돈했습니다. 활성화 시에는 연한 블루 배경(`rgba(var(--tertiary-rgb), 0.08)`) 및 토스 블루 글씨(`var(--tertiary)`)로 시각적으로 또렷하게 구분됩니다.
- **모바일/태블릿 가로 스크롤**: 화면 폭이 좁은 모바일 및 태블릿 디바이스에서 탭 메뉴가 잘리거나 깨지지 않고 부드럽게 옆으로 스와이프할 수 있도록 가로 스크롤 및 스크롤바 숨김 속성을 이식했습니다.

### 3. 토스 테크 블로그 레퍼런스 스타일 피드 카드
- **가독성 향상 타이포그래피**: 대한민국 금융/IT 서비스 표준 서체인 `Pretendard` 전역 서체를 최우선 바인딩하고, 자간을 `-0.018em`으로 쫀쫀하게 조율하고 `word-break: keep-all;`을 전역 적용하여 한글 문맥의 가독성을 최상으로 끌어올렸습니다.
- **매거진 룩 & 호버 모션**: 피드 카드의 간격을 `24px`로 넉넉하게 벌려 매거진 룩을 유도하고, 썸네일 이미지의 높이를 `200px`로 시원하게 배치했습니다. 마우스 호버 시 카드 본체가 Y축으로 가볍게 뜨며(`translateY(-4px)`) 썸네일 이미지가 `1.04` 비율로 줌인되는 모션 효과를 제공합니다.

### 4. 금융 앱 홈 스타일 사이드바 & 인라인 토글
- **내 자산 스타일 대시보드**: 사이드바의 통계(Stat) 블록을 토스 앱의 "내 자산" 컴포넌트처럼 리디자인하여, 각 통계 지표를 독립된 그레이 라운드 카드 블록으로 입히고 수치를 볼드한 토스 블루 컬러로 하이라이트했습니다.
- **인라인 토글 단추 & 에지 탭**: 사이드바 우측 하단의 플로팅 단추를 사이드바 하단 퀵링크 영역 우측으로 인라인 통합해 레이아웃을 정돈했습니다. 사이드바가 접히면 화면 우측 끝자리에 밀착된 돌출형 탭으로 자동 변환됩니다.
- **초기 노출 상태 활성화 (열림 기본)**: 로컬스토리지 초기 상태나 최초 페이지 진입 시 사이드바가 기본적으로 열린 채 제공되며, 접은 이력은 브라우저 스토리지에 세밀하게 보존됩니다.

### 5. 반응형 검색 드롭다운 & 캡슐형 검색바
- **플랫 캡슐형 검색 입력**: 검색바를 회색 타원 캡슐형(`background: #f2f4f6`)으로 교체하고 포커스 시 흰색 배경 반전과 함께 Toss Blue 아웃라인 효과가 활성화되도록 개편했습니다.
- **가로폭 동적 동기화**: 검색 드롭다운 결과 패널(`.search-menu-container`, `.menu-panel-results`)이 고정 폭 대신 메인 카테고리/글 목록(`.category-list`)의 너비와 완전히 일치하도록 반응형 계산식을 적용했습니다.
  - 데스크톱: 사이드바 유무에 따라 카테고리 목록의 너비(최대 1110px 또는 790px)와 1:1로 대응합니다.
  - 모바일/태블릿: 화면 좌우 20px씩 여백을 둔 꽉 찬 크기(`calc(100vw - 40px)`)로 유동 작동합니다.

### 6. 다이나믹 반응형 스페이싱 및 A11y
- **유기적 가변 간격**: 내비게이션 영역과 하단 카테고리 리스트 사이의 여백을 데스크톱 기준 **36px**로 넉넉히 주되, 태블릿에서는 **24px**, 모바일에서는 **16px**로 미디어 쿼리를 세분화하여 축소 조율되도록 반응형으로 제어했습니다.
- **웹 접근성(A11y) 보강**: 사이드바 토글 버튼에 다국어(ko/en) 연동 텍스트가 바인딩된 `aria-label` 속성을 매핑하여 스크린 리더 환경의 웹 접근성을 준수했습니다.

## :file_folder: 디렉토리 구조

```bash
toss-style-theme/
├── about.json            # 테마 정보, 토스 고유 컬러 스키마 매핑
├── settings.yml          # 테마 설정값 매핑
├── locales/              # 다국어 번역 파일 (ko, en)
├── scss/
│   ├── common/           # 공통 디자인 (글로벌 둥글기, 피드 카드, 인풋, 뱃지 등)
│   ├── desktop/          # 데스크톱/태블릿 2열 그리드 및 헤더, 사이드바, 검색 오버라이드
│   ├── mobile/           # 모바일 가로 스크롤 및 인풋 모바일에 특화된 스타일
│   └── var-mixin.scss    # TDS 변수 토큰 및 토스 호버/그림자 믹스인 정의
└── javascripts/
    └── discourse/
        ├── api-initializers/  # 사이드바 디폴트 활성화 및 모바일 변환 스크립트
        └── components/        # 토글 버튼 등 Ember GJS strict 모드 컴포넌트
```
