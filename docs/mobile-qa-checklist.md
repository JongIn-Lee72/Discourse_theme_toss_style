# 모바일 QA 체크리스트 (Mobile QA Checklist)

본 문서는 Toss Style Theme의 모바일 환경(320px ~ 768px)에서 시각적 일관성과 동작성을 완벽히 검증하기 위한 10대 핵심 QA 시나리오와 체크포인트를 정의합니다.

---

## 1. 환경 및 대상 기기 (Test Environment)
- **Viewport**: 320px (iPhone SE), 375px~390px (iPhone 13/14), 412px~430px (Galaxy S22/S23), 768px (iPad/Tablet Portrait)
- **Browsers**: iOS Safari (실기기 및 시뮬레이터), Android Chrome, Chrome DevTools 모바일 에뮬레이터
- **Theme Modes**: 라이트 모드 (Default), 다크 모드 (Toss Style - Dark)

---

## 2. 10대 핵심 QA 시나리오 (Key Test Scenarios)

### Scenario 1: 통합 검색바 알약형 배치 및 드롭다운 포커스
- **수행 방법**: 모바일 화면 상단에 재배치된 검색바(`.unified-search-bar`)를 터치하여 입력 상태로 전환합니다.
- **체크포인트**:
  - [ ] 검색창 터치 시 가상 키보드가 정상 노출되는가?
  - [ ] 검색 결과 드롭다운(`.search-dropdown-anchor`)이 화면 좌우 16px의 여백을 유지하며 짤림 없이 100% 폭으로 흐르듯 늘어나는가?
  - [ ] 드롭다운 내부 링크 터치 및 클릭 방해 요소가 없는가?
  - [ ] 키보드가 내려가거나 포커스를 잃었을 때 드롭다운이 정상적으로 닫히는가?

### Scenario 2: 하단 플로팅 UI 및 Safe-Area 대응
- **수행 방법**: 홈 화면, 카테고리 목록, 토픽 목록 등을 맨 아래까지 끝까지 스크롤합니다.
- **체크포인트**:
  - [ ] 모바일 하단 플로팅 네비게이션바(`#navigation-bar`)와 글쓰기 FAB(`#create-topic`) 버튼이 기기의 하단 바(iOS Home Indicator 등)와 겹치지 않고 safe-area 여백을 확보하는가?
  - [ ] 스크롤을 맨 아래로 내렸을 때, 마지막 목록 아이템이 하단 플로팅 버튼 뒤에 가려져 터치할 수 없는 문제가 완벽히 해소되었는가? (목록 하단에 `120px` 이상의 충분한 패딩이 확보되는가?)

### Scenario 3: nav-pills 탭 메뉴 가로 스크롤 및 터치 스와이프
- **수행 방법**: 카테고리나 태그 필터링 등 탭이 많은 화면에서 탭 메뉴 영역을 좌우로 스크롤해봅니다.
- **체크포인트**:
  - [ ] 탭이 여러 줄로 깨져서 줄바꿈되지 않고, 단일 행 가로 방향(`row nowrap`)으로 배치되는가?
  - [ ] 터치 슬라이드(스와이프)로 부드럽게 좌우 스크롤이 가능한가?
  - [ ] 스크롤바가 화면상에 노출되지 않아 시각적으로 깔끔한가? (`scrollbar-width: none` 등 적용 확인)

### Scenario 4: 반응형 피드 카드(토픽 목록) 레이아웃 및 Overflow 방지
- **수행 방법**: 320px(iPhone SE 등) 가장 좁은 뷰포트에서 토픽 목록 피드를 스크롤해봅니다.
- **체크포인트**:
  - [ ] 개별 토픽 카드(`.main-link`)에 `max-inline-size: 100%` 및 `box-sizing: border-box`가 적용되어 화면 밖으로 넘치지 않는가?
  - [ ] 브라우저에 불필요한 가로 스크롤바가 나타나지 않는가?
  - [ ] 썸네일 이미지 높이가 `150px` 이하로 모바일에 맞게 컴팩트하게 조절되는가?

### Scenario 5: 카테고리 카드 뷰의 시각적 청소 (TDS ListRow 정렬)
- **수행 방법**: 카테고리 전체 목록 페이지(`categories-list`)에 진입합니다.
- **체크포인트**:
  - [ ] 카테고리 카드 내부의 제목(`! [공통] 공지사항` 등)이 흰색 중복 버블/그림자 상자 안에 갇혀 있지 않고, 투명하고 플랫한 텍스트로 깔끔하게 렌더링되는가?
  - [ ] 카테고리 카드 왼쪽의 고유 색상 바(두께 6px 가량)와 카드의 둥근 모서리(`--tds-radius-l`)가 조화롭게 어우러지는가?

### Scenario 6: 토픽 상세 본문(포스트 스트림) 위계 및 가독성
- **수행 방법**: 임의의 토픽 게시글 상세 페이지에 진입하여 글 내용을 읽어봅니다.
- **체크포인트**:
  - [ ] 포스트 카드 본문(`.topic-body`)의 상하좌우 패딩이 `--tds-space-l`(16px)로 적절히 분배되어 가독성이 좋은가?
  - [ ] 본문의 줄 높이(`line-height: 1.6`)가 넉넉히 확보되어 줄 간격이 빽빽해 보이지 않는가?
  - [ ] 작성자 아바타(`45px` 크기)와 이름, 날짜 텍스트 정렬이 깨지거나 어긋나지 않는가?

### Scenario 7: 글쓰기 에디터(Composer) 슬라이드 및 입력 방해 차단
- **수행 방법**: 글쓰기 버튼을 눌러 에디터를 열고 입력을 진행합니다.
- **체크포인트**:
  - [ ] 모바일 키보드가 올라왔을 때 에디터 입력 창이 화면에 다 가려지지 않고 정상 스크롤/포커스 이동되는가?
  - [ ] 에디터 최소/최대화 버튼 및 닫기 버튼이 모바일 손가락 터치 영역에 맞춰 충분한 크기를 갖는가?

### Scenario 8: 특수 페이지별 Grid Reset 레이아웃 검증
- **수행 방법**: 아래의 페이지들에 모바일 기기로 각각 진입하여 레이아웃을 확인합니다.
  - *대상*: 프로필 페이지, 사용자 활동(Activity) 탭, 쪽지함(PM), 검색 결과 페이지, 문서(Docs) 페이지, 관리자(Admin) 페이지
- **체크포인트**:
  - [ ] 데스크톱 전용 2단/3단 그리드가 해제되어 모바일에 맞게 세로 1단(`display: block` 또는 `flex-direction: column`)으로 단일하게 흐르는가?
  - [ ] 특정 레이아웃이 겹치거나 일부 버튼이 화면을 이탈하는 증상이 없는가?

### Scenario 9: 모바일 Touch Active 상태 처리 (Stuck Hover 방지)
- **수행 방법**: 토픽 카드 또는 카테고리 탭을 터치(Tap)했다가 뗍니다.
- **체크포인트**:
  - [ ] 마우스를 사용하는 데스크톱 호버 모션(Scale 확대 등)이 모바일 터치 후에도 해제되지 않고 멈춰 있는(stuck hover) 현상이 제거되었는가?
  - [ ] 터치한 짧은 순간에만 즉각적인 터치 피드백(`.active` 등)이 발생하고 손가락을 떼면 원상복구되는가?

### Scenario 10: Toss Style Dark Mode 테마 호환성
- **수행 방법**: 우측 상단 햄버거 메뉴나 개인 설정에서 다크 모드 테마를 활성화합니다.
- **체크포인트**:
  - [ ] 모바일용 커스텀 스타일 및 탭 바에 하드코딩된 밝은 색상(#ffffff 등)이 남아 있어 다크 모드 전환 시 하얗게 뜨는 버그가 없는가?
  - [ ] 텍스트와 배경의 대비가 최소 4.5:1 이상을 충족하여 야간 가독성을 보장하는가? (모든 색상이 `--color-text`, `--color-surface` 등 시맨틱 변수로 치환되었는지 확인)
